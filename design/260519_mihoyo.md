# 米哈游招聘交流会

## Archi（关卡策划）

- 什么是关卡策划：白盒设计、玩法设计、任务设计、叙事设计等
	- 画总平面
	- 规划主动线：引导与“藏”
	- 主题玩法设计
	- 任务设计
- 案例demo：盗墓倒斗玩法
	- 设计背景：整体主线&关卡背景
	- 设计目标：动态关卡/有趣玩法/震撼表演/复用3C（交互，如“按F交互”）
	- ==设计切入点==：平面、剖面、造型等
		- 震撼观景点
		- 关键桥段
		- 叙事故事
	- ==规则设定==：明确核心玩法后确定交互规则
		- 区分逻辑，规则传达清晰：特别材质（彩色）、不可破坏结构（灰色）、可攀爬点（绿色）
	- ==发散玩法==：围绕核心玩法拓展
		- 不新增复杂机制
		- 结合现实常识：如复用球形炸弹使其滚动
	- ==各种点缀==（丰富体验）：“变奏”
- “建筑设计”与“关卡策划”的关联
	- 空间的功能：基于“题”定功能
	- 动线设计
	- 规划&布局
	- 空间形态&尺度：空间感受决定空间形态
	- 视线
- “关卡设计”与“建筑设计”的区别
	- 出发点不同：新鲜感、激发好奇心、是否有趣 而非 合理、好用与功能
	- 人的行动方式不同：可以决定出生点，而非入口和出口，需要考虑爬墙/飞翔等情况
	- 受限情况不同：不受现实合理性约束（不需要上厕所），四维空间传递故事
	- 向外沟通的内容不同
	- 规则的传达方式不同：类似机械使用说明书
	- 项目反馈周期不同
	- 情绪反馈来源不同
- 优势
	- 未来大世界需求多
	- 设计表达能力——草图能力、白盒搭建能力
	- 设计思维：目标导向、从体验者角度出发、空间感知能力
	- 审美能力：接触关卡美术、原画部门，审美上快速达成一致
- 建筑转策划，需要什么
	- 关卡搭建能力
	- 逻辑能力：类似导演，调动积极性
	- 软件能力：让场景动起来，建议学UE
	- 生活采风：拆解关卡，临摹+魔改
	- 设计灵感：多去旅游，去重庆感受“魂系关卡”
	- 增加积累：游戏体验，深度+广度；电影、动画、漫画
	- 能力迁移：多学科知识迁移（物理、机械、历史、美术……） → 重构世界（如《辐射》改能源）；建筑落地技能提升（Rhino/SU建模+AI/PS）
	- 动手实操，准备作品集：==10个demo计划==
		- 介绍游戏经历，体现个人理解
		- 空间逻辑清晰
		- 流程安排合理
		- 有“建筑味儿”的创新
		- 附“设计笔记”
- 思考
	- 设计这个会晕3D……

## Cissie（场景美术）

> 提瓦特互动建筑

- 跨学科的回归：学科是后设，跨学科是一种回归。
- 提瓦特建筑语料：用城规思路推演人文布局、用符号概念统筹复杂信息、按功能动线规划玩法空间
	- **用城规思路推演人文布局**
		- ==artbook==：自然风貌、人文风貌，确认设计概念方向
		- ==建筑拆分&景观拆分==
		- poi：特殊事件、重要地点——从导演的角度（分镜）来设计场景 → ==情绪板（moodboard）==沟通
	- **用符号概念统筹复杂信息**：“集体无意识”锚点 → “阶级与秩序的外化”
		- ==建筑符号化== → IP设计
	- **按功能动线规划玩法空间**：世界构建与生态群落推演、玩法片区与动线引导、POI布置
		- 世界构建与生态群落推演：风貌及水系推演、生态群落分布
		- 功能片区与动线引导：建筑单体设计 → ==建筑模块化拆分==、功能气泡图
		- 资产规划与模块化设计：推敲一堵墙4×4还是15×8更能复用，如何让基础模块互相穿插，共用材质贴图把渲染批次降到最低
		- 视觉牵引与核心交互：遵从空间法则，设计视线锚点，参考园林设计
- 整点儿好玩的！——脱掉物理躯壳，做灵魂的造梦机器。
- 思考
	- 这个好玩！可以试着做几套设计（先分析既有），从整体到立面到细节（比如给明日方舟做同人场景设计，整理设定→moodboard→平面→空间→立面→细部）
	- 建筑史可以作为一种认知积淀
	- BIM似乎可以迁移为一种建模流程设计的能力
	- 虽然这样可以继续发展建筑能力，但我的建筑基础能力可能比不过建筑本科的同学
